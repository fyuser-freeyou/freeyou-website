1. Complete Data Flow — From Lead to Verified Professional

This is the full journey a person takes from first landing on the FreeYou website to becoming a KYC-verified freelancer on the platform. Each block below is clickable — tap to see what happens inside that component.

Step 1 — Website
Lead fills in the early access form on freeyou.co.in
The person enters their name, phone, email, skill, and city. When they click "Apply", the browser sends a POST request to the Cloudflare Worker — not directly to Zoho (avoids CORS issues).
POST https://freeyou.co.in/api/submit-lead Body: { "First Name": "Arjun", "Phone": "+919876543210", "Email": "...", "Description": "Web Development", ... }
Step 2 — Cloudflare Worker
Worker receives the data and forwards it to Zoho CRM
The Worker (_worker.js) receives the JSON, converts it to a FormData payload that Zoho's WebToLead endpoint expects, and submits it. This is invisible to the user — they just see a success message.
File: /FreeYou/_worker.js → function handleLeadSubmit() Sends to: https://crm.zoho.in/crm/WebToLeadForm (POST) Tokens: ZOHO_TOKEN_1, ZOHO_TOKEN_2 (hardcoded in Worker)
Step 3 — Zoho CRM
Lead record created in Zoho CRM → Welcome email triggered
A new "Lead" record appears in Zoho CRM. An automated workflow rule fires: if a new lead is created, send a welcome email. This is the "Rule 1" workflow Ronald created manually in Zoho.
Zoho CRM → Setup → Automation → Workflow Rules Rule: "Welcome Email on Lead Created" Action: Send email from hello@freeyou.co.in
Step 4 — Manual Review
Team qualifies the lead in Zoho and changes status to "Qualified"
Someone on the FreeYou team reviews the lead, has an intro call, and changes the Lead Status field to "Qualified". This status change is the trigger for the KYC process to start.
Zoho CRM → Lead record → Lead Status field → "Qualified" This triggers: Workflow Rule 2 → Webhook to n8n
Step 5 — n8n Automation
Zoho fires a webhook to n8n, which creates a KYC session
Zoho's workflow rule sends the lead's data to an n8n webhook URL. n8n extracts the name, email, phone, then calls the Didit API to create a unique KYC verification session for that person.
File: /FreeYou/freeyou-n8n-didit-workflow.json n8n Webhook: POST https://[n8n-url]/webhook/freeyou-kyc Calls: POST https://verification.didit.me/v3/session/
Step 6 — KYC Email
n8n emails the lead a secure Didit verification link
n8n sends a branded HTML email to the lead with a "Start KYC Verification" button. The link is unique to their session and expires in 24 hours. It simultaneously updates the lead's status in Zoho to "KYC Sent".
n8n Email node: SMTP via hello@freeyou.co.in Link: https://verify.didit.me/s/[unique-session-id] Zoho update: Lead_Status → "KYC Sent", KYC_Session_ID saved
Step 7 — Didit KYC
Lead completes identity verification on Didit's hosted page
The lead clicks the link, lands on Didit's hosted verification page, uploads their Aadhaar/PAN/Passport and takes a selfie. Didit runs liveness detection and document verification. Takes 2–3 minutes.
Didit hosted at: https://verify.didit.me/s/[session-id] Accepts: Aadhaar, PAN, Passport, Driving Licence Liveness: Yes (face match required)
Step 8 — Webhook Result
Didit sends the verdict to FreeYou's webhook endpoint
After verification, Didit sends a signed webhook to the Cloudflare Worker at /api/didit-webhook. The Worker verifies the HMAC signature (security), then updates the lead's KYC fields in Zoho CRM — Approved or Declined.
Webhook: POST https://freeyou.co.in/api/didit-webhook File: /FreeYou/_worker.js → function handleDigitWebhook() Zoho update: KYC_Status, KYC_Verified_At, Lead_Status

4. Configuration — All Keys & Where They Live

These are the secret keys that make the platform work. They are stored in Cloudflare Worker environment variables — never in the code files themselves. To update them: Cloudflare Dashboard → Workers → freeyou-worker → Settings → Variables.

ZOHO_REFRESH_TOKEN
Zoho OAuth token for the Worker to update CRM records via API
📍 Cloudflare Worker secrets
ZOHO_CLIENT_ID
Zoho API app client ID (from Zoho API Console)
📍 Cloudflare Worker secrets
ZOHO_CLIENT_SECRET
Zoho API app client secret
📍 Cloudflare Worker secrets
DIDIT_API_KEY
Didit API key for creating KYC sessions
📍 Cloudflare Worker secrets + n8n credential
DIDIT_WEBHOOK_SECRET
Used to verify that webhook calls are genuinely from Didit (security)
📍 Cloudflare Worker secrets
DIDIT_WORKFLOW_ID
The ID of your Didit verification workflow (which documents to accept)
📍 n8n variable (Settings → Variables)
ZOHO_TOKEN_1 / TOKEN_2
Zoho WebToLead tokens. Currently hardcoded in _worker.js — move to env vars for security
⚠️ Currently in _worker.js (lines 11–12)

6. Future Additions — How to Stitch Them In

Each addition below is designed to plug into the existing system without rebuilding anything. The existing lead → CRM → KYC pipeline is the foundation.

V2

Admin Dashboard

A password-protected page that reads from Zoho CRM API to show all leads, their KYC status, and actions (approve/reject). Built as a separate HTML page calling a new Worker endpoint.

🔗 Requires: Zoho CRM API access (already set up) · New Worker route: /api/admin/leads
V2

Freelancer Profile Pages

Public profile pages for verified freelancers. Stored in Zoho CRM (converted from Lead to Contact after KYC), displayed as /profile/[slug] — Worker reads the Contact record and renders the page.

🔗 Requires: KYC flow complete · Zoho Contacts module · New Worker route: /profile/[slug]
V2

Team Builder Feature

A page where clients can browse verified freelancers by skill and city, and request an introduction. Form submission goes to a new Worker endpoint, which creates a "deal" in Zoho CRM linking the client to the freelancer.

🔗 Requires: Freelancer profiles complete · New: Client lead module in Zoho · Worker route: /api/connect
V3

Daily Blog Content Agent

An n8n workflow that runs daily: fetches trending keywords from Google Search Console API → sends to Claude API → generates a blog post → saves to a /blog/ folder via GitHub API → triggers Cloudflare Pages redeploy.

🔗 Requires: Google Search Console API · Anthropic API key · GitHub repo connected to Cloudflare Pages
V3

Payment & Subscription

Razorpay integration for freelancer subscriptions. New Worker route /api/payment → Razorpay webhook → update Zoho subscription fields → email confirmation.

🔗 Requires: Razorpay account · New Worker route: /api/payment · Zoho subscription custom fields
